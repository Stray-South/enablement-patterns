# Pattern 08: Self-Serve Documentation Hubs

> **Status: Stubbed.** Full pattern coming in Week 4. See `PLANS.md`.

## Anchor

The pattern that closes the Four-Stage Engagement Pipeline (Pattern 01). How do clients sustain enablement after Handoff? Through documentation that the team uses to answer their own questions, reducing the support load on the Enablement function while increasing the team's operational independence.

The specific approach: identify the questions the team is most likely to ask in the first 90 days post-Handoff, build the documentation that answers those questions in the format the team will actually use, and measure the reduction in direct support requests as the primary success metric. This is documentation as a feature of the deployed system, not as an artifact handed over at the end.

The methodology that produced a 40% reduction in technical support requests across multiple programs (Georgia Tech 2021-2023, University of Montevallo 2023-2025) is the methodology this pattern documents. It is not new. It is rigorous application of instructional design principles to AI system documentation, with measurement attached.
